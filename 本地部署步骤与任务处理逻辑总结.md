## 二、机器人端部署步骤

### 2.1 硬件准备
- 机器人：宇树 Go2，固件升级到最新版
- 运动模式：通过手机 App 切换为 **AI 模式**（降低躯干抖动，提高视觉稳定性）
- 相机：Intel RealSense D455（推荐）/ D435，使用 USB 3.2 Gen1 线缆连接算力盒前面板 Type-C 口
- 通信：默认网段 `192.168.123.x`，网卡配置为 `eno1`（本地配置）

### 2.2 软件环境安装

```bash
# 1. 安装 Unitree SDK2 Python
cd ~/unitree_sdk2_python
pip3 install -e .

# 2. 安装 RealSense 库
pip3 install pyrealsense2

# 3. 安装 InternNav 依赖
cd ~/unitree_sdk2_python/InternNav
pip3 install numpy requests Pillow casadi

# 4. 如遇到 CycloneDDS 缺失，需手动编译安装
```

### 2.3 启动流程（一键启动）

执行 `InternNav-deploy/onboard/start_robot.sh`，脚本按顺序启动 **4 个模块**：

| 启动顺序 | 模块脚本 | 功能 |
|---|---|---|
| [1/4] | `unitree_driver.py` | 底层运动控制驱动，UDP :8899 接收指令，遥控器优先 |
| [2/4] | `ros_bridge.py` | ROS2 桥接：`/cmd_vel_bridge` → UDP，`/utlidar/robot_odom` → `/odom_bridge` |
| [3/4] | `go2_camera_publisher.py` | Go2 原生相机 RGB 图像发布 + 虚拟深度图（以疏通时间同步器） |
| [4/4] | `sport_state_publisher.py` | 从 DDS `rt/lf/sportmodestate` 获取位姿，发布 `/odom_bridge`（替代 LiDAR odom） |

> 注：原版 InternNav-deploy 文档使用 `realsense_publisher.py`（RealSense 深度相机），本地配置使用 `go2_camera_publisher.py`（Go2 原生相机 + 合成深度图）。

### 2.4 导航客户端启动

在另一个终端执行：

```bash
cd ~/unitree_sdk2_python/InternNav
python3 scripts/realworld/http_internvla_client.py
```

### 3.2 各模块详细逻辑

#### (1) `go2_camera_publisher.py` — 图像采集与发布

- 通过 Unitree SDK `VideoClient` 从 Go2 原生相机采集 RGB 图像
- 发布话题：
  - `/camera/camera/color/image_raw` — RGB 图像 (sensor_msgs/Image)
  - `/camera/camera/aligned_depth_to_color/image_raw` — 合成深度图（固定 2.0m 常数值，uint16）
  - `/camera/camera/color/camera_info` — 相机内参
- **合成深度图的原因**：`http_internvla_client.py` 使用 `ApproximateTimeSynchronizer` 同步 RGB+深度，若没有深度话题则同步器永不触发回调

#### (2) `sport_state_publisher.py` — 里程计发布

- 从 DDS 话题 `rt/lf/sportmodestate` 订阅 Go2 运动状态（位姿、速度、IMU 姿态）
- 转换为 `nav_msgs/Odometry` 发布到 `/odom_bridge`
- **存在原因**：Go2 没有激光雷达 SLAM 时 `/utlidar/robot_odom` 不存在，原 `ros_bridge.py` 依赖该话题产生 `/odom_bridge` 会失效

#### (3) `ros_bridge.py` — ROS2 消息桥接

- **下行**：订阅 `/cmd_vel_bridge` (Twist) → 打包为二进制 (vx, vyaw) → UDP 发送到 `127.0.0.1:8899`
- **上行**：订阅 `/utlidar/robot_odom` → 转发到 `/odom_bridge`（当有 LiDAR 时）

#### (4) `unitree_driver.py` — 底层运动控制

- 初始化 Unitree SDK，建立 DDS 通信
- 订阅 `rt/lowstate` 获取遥控器状态
- UDP 监听 `127.0.0.1:8899`，接收 (vx, vyaw) 速度指令
- **优先级逻辑**：
  - 遥控器摇杆动作 > UDP 脚本指令
  - 遥控器活动时立即释放控制权，脚本暂停
  - UDP 超时 1.0s 自动归零速度（安全看门狗）
- 通过 `SportClient.Move(vx, 0, vyaw)` 控制机器人
- 循环频率 ~25Hz

#### (5) `http_internvla_client.py` — 导航客户端主程序

**Go2Manager ROS2 节点**：
- 使用 `ApproximateTimeSynchronizer` 同步 RGB + Depth 话题（slop=0.1s）
- 订阅 `/odom_bridge` 获取里程计
- 发布 `/cmd_vel_bridge` 速度指令
- 将 RGB 图像转 JPEG、深度图转 PNG 存储为 bytes

**planning_thread（规划线程）**：
- 等待新图像到达 (`manager.new_image_arrived`)
- 找到与图像时间戳最接近的里程计数据
- 调用 `dual_sys_eval()` 发送 HTTP POST 到推理服务器
  - 请求体：RGB JPEG + Depth PNG + JSON（reset 标志、当前速度）
  - 响应：`discrete_action`（离散动作）或 `trajectory`（连续轨迹）
- **离散动作模式**：调用 `manager.incremental_change_goal()` 累积改变目标位姿，切换到 PID 控制
- **连续轨迹模式**：将轨迹从机器人坐标系转换到世界坐标系，更新 MPC 参考轨迹，切换到 MPC 控制

**control_thread（控制线程）**：
- 以 10Hz 频率运行
- **MPC 模式**：以当前里程计为初始状态，求解 MPC 优化问题，输出 (v, w)，通过 `manager.move()` 发布
- **PID 模式**：计算 homography 位姿误差，PD 控制输出 (v, w)

#### (6) `controllers.py` — 控制器实现

- **Mpc_controller**：基于 CasADi + IPOPT 的非线性 MPC
  - 预测时域 N=20，控制周期 T=0.1s
  - 代价函数：追踪参考轨迹 + 控制量惩罚
  - 约束：v ∈ [0, 0.4], w ∈ [-0.4, 0.4]
  - 每次求解使用上一时刻的最优解作为热启动
  - 参考轨迹自动从全局规划路径中按弧长选取最近点

- **PID_controller**：平移+偏航 PD 控制器
  - Kp_trans=2.0, Kd_trans=0.0, Kp_yaw=1.5, Kd_yaw=0.0
  - 最大速度 v_max=0.6, w_max=0.5
  - 前进速度为负时钳位为 0（不允许后退）

#### (7) `http_internvla_server.py` — 推理服务器

- Flask HTTP 服务，监听 `0.0.0.0:5801`
- 路由 `POST /eval_dual`：
  - 接收 RGB 图像 + 深度图像 + JSON（含 reset 标志、当前 v/w）
  - 加载 InternVLA-N1 模型（`checkpoints/InternVLA-N1`）
  - 调用 `InternVLAN1AsyncAgent.step()` 进行双系统推理
  - 返回 `discrete_action` 或 `trajectory` + `pixel_goal`

#### (8) `internvla_n1_agent_realworld.py` — VLA 导航智能体

- **step_s2（高层规划，System-2）**：
  - 将当前 RGB 图像 + 历史图像输入 VLA 多模态大模型
  - 注入当前速度状态信息（静止/前进/后退 + 转向状态）
  - 模型输出推理结果：
    - 含数字坐标 → 解析为像素目标 (x, y)，生成 latent，后续进入 S1 轨迹生成
    - 含方向箭头 (↑←→↓/STOP) → 解析为离散动作序列
  - 使用 KV-cache 加速连续推理
  - 每 `PLAN_STEP_GAP` 步或被要求 look_down 时重新推理

- **step_s1（低层轨迹生成，System-1）**：
  - 根据 S2 输出的 latent + 当前/目标 RGB-D 图像
  - 调用 `model.generate_traj()` 生成局部运动轨迹

## 四、本地修改 vs 上游仓库差异总结

| 组件 | 上游 (InternNav-deploy) | 本地修改 |
|---|---|---|
| 相机节点 | `realsense_publisher.py` (RealSense D455) | 新增 `go2_camera_publisher.py`，使用 Go2 原生相机 + 合成深度图 |
| 启动脚本 | 启动 3 个模块 | 改为 4 个模块：增加 `sport_state_publisher.py` 和 `go2_camera_publisher.py` |
| 里程计来源 | 依赖 `/utlidar/robot_odom` 话题 | 新增 `sport_state_publisher.py` 从 DDS `rt/lf/sportmodestate` 直接获取 |
| 网卡配置 | `eth0` | 改为 `eno1` |
| 推理地址 | 默认 `127.0.0.1:5801` | 使用 NAT 穿透地址连接远程 GPU 服务器 |

本地 `unitree_sdk2_python` 目录相比上游官方仓库多出：
- `InternNav/` — 完整的 InternNav 导航框架
- `InternNav-deploy/` — 机器人端部署配置
- `go2_teleop_lerobot_tools/` — 遥操作工具
- `*.md` / `*.pdf` — 本地文档和调试记录
- `go2_teleop_lerobot_tools.zip` — 工具压缩包

### 5.4 注意事项

1. **大文件问题**：模型检查点文件（`checkpoints/InternVLA-N1`）通常很大，不要推送到 GitHub。确保 `.gitignore` 中包含：
   ```
   checkpoints/
   *.pth
   *.bin
   *.safetensors
   outputs/
   test_data/
   ```

2. **子目录的 `.git`**：`InternNav/` 和 `InternNav-deploy/` 各带自己的 `.git`，如果要把它们作为主仓库的一部分推送，需要删除它们的 `.git` 目录。

3. **敏感信息**：`http_internvla_client.py` 中包含 NAT 代理地址等配置，确认没有暴露敏感凭据。

4. **分支选择**：GitHub 默认分支名可能是 `main` 而非 `master`，推送前确认分支名。
